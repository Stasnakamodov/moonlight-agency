# Задача: Сделать Млечный Путь ЖИВЫМ

## Контекст проекта

Сайт космического агентства (`moonlight-agency`) с полноэкранным canvas-фоном звёздного неба.
Весь рендеринг — в одном компоненте: `src/components/effects/StarField.tsx` (~1300 строк, Canvas 2D API, React useEffect).

Стек: **Next.js 15 + React 19 + TypeScript + Tailwind CSS 4**.

## Что уже есть

Файл `StarField.tsx` содержит:
- **Perlin noise** + **fbm** + **domain warping** (строки 1-73)
- **Milky Way band** — статическая текстура, pre-rendered в offscreen canvas `nebulaBase` (функция `generateBaseNebula`, строки ~498-561). Линия MW задана уравнением: `d = abs(-0.65*nx - ny + 0.82) / 1.192`, ширина 0.22, профиль `t*t`
- **Zone nebulae** — domain-warped текстуры зон, рисуются аддитивно поверх
- **7 слоёв звёзд** с мерцанием (scintillation), цветами по спектральному классу Morgan-Keenan
- **Deep field** — тысячи субпиксельных точек фона
- **Dust particles** — летающие пылинки переднего плана
- **Shooting stars** — падающие звёзды
- **Galaxies** — далёкие галактики
- **Параллакс** мыши и скролла
- Анимационный цикл `loop()` с `driftTime` (накапливаемые секунды), `smoothMouseX/Y`, `smoothScroll`
- `reduced` motion support

**Проблема:** Сам Млечный Путь — самый крупный визуальный элемент — полностью СТАТИЧЕН. Текстура `nebulaBase` рисуется один раз и больше не меняется. Это выглядит мёртво.

Предыдущая попытка добавить "дыхание" (globalAlpha sine wave) и "дрейфующие светящиеся пятна" (radial gradient патчи) — не дала визуально заметного результата.

## Что нужно сделать

Сделать Млечный Путь **ЖИВЫМ** — чтобы при взгляде на экран было ощущение, что газовые облака медленно движутся, переливаются, дышат. Как в реальных таймлапсах ночного неба.

### Требования:
1. **Визуально заметный эффект** — не 4% opacity change, а реально видимое движение/пульсация газовых облаков
2. **Плавное, медленное, органичное** — не мигание, а именно "живое дыхание" как у настоящей туманности
3. **Только Canvas 2D API** — никаких WebGL, шейдеров, дополнительных библиотек
4. **Вписаться в бюджет кадра** — текущий рендер ~2-3ms, бюджет до 16ms. Можно использовать до ~8-10ms
5. **Не ломать существующую архитектуру** — модифицировать только `StarField.tsx`
6. **Reduced motion** — при `prefers-reduced-motion: reduce` всё отключается

### Направления для исследования:
- Как создают анимированные nebula/milky way эффекты на canvas 2D
- Техники: animated noise textures, layered offscreen canvases with moving blend, temporal noise offset, realtime fbm с drifting offset, multi-layer parallax nebula strips
- Может быть стоит не пытаться анимировать единственную статичную текстуру, а пересобрать подход — например, рендерить несколько слоёв MW с разными noise offsets и анимировать их взаимное наложение
- Или использовать realtime noise sampling (не для всей текстуры, а для ограниченного числа "облачных слоёв" поверх базы)

### Что точно НЕ работает:
- `globalAlpha` пульсация на ±4-12% — незаметно
- Статические radial gradient патчи с alpha 0.01-0.12 поверх — незаметно
- Всё, что опирается только на прозрачность без реального визуального движения/деформации

## Порядок работы

1. **Изучи файл** `src/components/effects/StarField.tsx` полностью — пойми архитектуру рендеринга
2. **Изучи в сети** реальные референсы: как выглядит анимированный Milky Way, какие техники используют для canvas 2D nebula animation, living space backgrounds
3. **Войди в режим планирования** (EnterPlanMode) — продумай конкретный план реализации
4. **Реализуй** после одобрения плана
5. **Проверь визуально** — эффект должен быть ОЧЕВИДНО виден невооружённым глазом
