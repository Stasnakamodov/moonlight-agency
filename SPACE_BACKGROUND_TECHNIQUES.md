## 1. Звёзды и звёздные поля

### 1.1 WebGL Instanced-частицы звёзд (рекомендуемый апгрейд)

**Что это:** Тысячи звёздных точек, рендерящихся через WebGL с использованием `THREE.InstancedMesh` или `THREE.Points` с `BufferGeometry`. Каждая звезда имеет индивидуальную позицию, цвет, размер и фазу мерцания — всё вычисляется на GPU.

**Как реализовать:**
- Использовать `@react-three/fiber` + `@react-three/drei` компонент `<Stars />` как базу, или написать свой `shaderMaterial`
- Создать `Float32Array` буферы для позиции (x,y,z), цвета (r,g,b), размера и фазы мерцания
- Распределить звёзды на большой сфере или кубе вокруг камеры
- Использовать кастомный вершинный шейдер для аттенуации размера: `gl_PointSize = size * (1.0 / -viewPosition.z)`
- Фрагментный шейдер создаёт мягкие круглые точки: `float d = distance(gl_PointCoord, vec2(0.5)); float alpha = 1.0 - smoothstep(0.0, 0.5, d);`
- Анимировать мерцание полностью в шейдере через `sin(uTime * speed + phase)` — ноль нагрузки на CPU
- Для 15 000+ звёзд использовать `THREE.InstancedMesh` с плоской геометрией и предвычисленными кватернионами для биллбординга (подход Richard Fu — 60+ FPS, ноль CPU-нагрузки на кадр)

**Реалистичное распределение цветов звёзд (звёздная классификация):**
```
O-тип (0.003%): #9bb0ff — голубовато-белые, самые горячие
B-тип (0.13%):  #aabfff — голубовато-белые
A-тип (0.6%):   #cad7ff — белые с голубым оттенком
F-тип (3%):     #f8f7ff — жёлто-белые
G-тип (7.6%):   #fff4ea — жёлтые (как наше Солнце)
K-тип (12.1%):  #ffd2a1 — оранжевые
M-тип (76.5%):  #ffcc6f — оранжево-красные, самые холодные, САМЫЕ РАСПРОСТРАНЁННЫЕ
```
Взвешивайте случайный выбор цвета СИЛЬНО в сторону M/K типов для реалистичности. Только 1-2 звезды должны быть яркими голубыми O/B типа.

**Визуальный эффект:** Высокий
**Стоимость для GPU:** Низкая (всё на GPU, 15к звёзд при 60 FPS доказано)
**Сложность:** Умеренная (кастомный шейдер) / Простая (drei `<Stars />`)
**Лучше всего для:** Основной слой звёздного поля — фундамент всего

**Ключевые ссылки:**
- drei `<Stars />`: `<Stars radius={100} depth={50} count={5000} factor={4} saturation={0} fade speed={1} />`
- Instanced starfield Ричарда Фу: https://www.richardfu.net/solving-starfield-perspective-distortion-in-3d-space-a-three-js-case-study/
- GPU-частицы Максима Хекеля: https://blog.maximeheckel.com/posts/the-magical-world-of-particles-with-react-three-fiber-and-shaders/

### 1.2 Canvas 2D многослойное звёздное поле (текущий подход — путь улучшения)

**Что это:** Ваш текущий подход. Несколько canvas-слоёв со звёздами разного размера, прозрачности и скорости дрейфа для имитации глубины.

**Как улучшить текущую реализацию:**
- **Добавить реалистичное распределение цветов** — сейчас только 4 голубых/белых варианта. Добавить тёплые оранжевые и жёлтые по звёздной классификации выше
- **Улучшить мерцание** — текущая синусоидальная прозрачность слишком однородная. Реальная сцинтилляция хроматическая: звёзды кратковременно смещаются между цветами. Реализовать варьированием RGB-каналов на чуть разных частотах: `r = base.r * (0.8 + 0.2 * sin(t * 1.1 + phase)); g = base.g * (0.8 + 0.2 * sin(t * 1.3 + phase)); b = base.b * (0.8 + 0.2 * sin(t * 0.9 + phase));`
- **Предрендер на offscreen canvas** — отрисовать статичные позиции звёзд один раз на offscreen canvas, затем `drawImage()` каждый кадр. Перерисовывать только оверлей мерцания
- **Использовать `{ alpha: false }` контекст** — устраняет накладные расходы альфа-композитинга для базового слоя
- **Целочисленные координаты** — `Math.floor()` или `Math.round()` позиций для избежания суб-пиксельного сглаживания

**Визуальный эффект:** Средний (с улучшениями: Средний-Высокий)
**Стоимость для GPU:** Низкая
**Сложность:** Простая
**Лучше всего для:** Фоллбэк для устройств без WebGL, или если хотите избежать зависимости от Three.js

### 1.3 CSS-only звёзды (массивы box-shadow)

**Что это:** Прозрачный элемент 1px с сотнями значений `box-shadow`, задающих позиции звёзд. Несколько элементов разного размера создают параллакс-слои.

**Как реализовать:**
- SASS-цикл `@for` генерирует случайные значения `box-shadow` во время сборки (ноль runtime-стоимости)
- 3 слоя: крошечные/тусклые (2000 теней, 1px), средние (500 теней, 2px), яркие (50 теней, 3px)
- Анимировать через `transform: translateY()` (GPU-композитинг) — НИКОГДА не анимировать само свойство box-shadow
- Использовать `vw`/`vh` единицы для адаптивного позиционирования
- Удвоить высоту массива теней и анимировать translateY от 0 до -50% для бесшовной бесконечной прокрутки

**Визуальный эффект:** Низкий-Средний
**Стоимость для GPU:** Ничтожная (статичная отрисовка, GPU-анимация трансформа)
**Сложность:** Простая
**Лучше всего для:** Максимально лёгкие страницы, или как самый глубокий/тусклый фоновый слой за WebGL-звёздным полем

**Ключевая ссылка:** https://codepen.io/saransh/pen/BKJun (классические параллакс-звёзды)

### 1.4 Алгоритмы мерцания / сцинтилляции звёзд

**Что это:** Создание реалистичного мерцания звёзд. Настоящая сцинтилляция вызвана атмосферной турбулентностью, преломляющей свет — это НЕ просто пульсация прозрачности.

**Как реализовать — три уровня реалистичности:**

**Уровень 1 — Вариация прозрачности (текущий, базовый):**
```
opacity = base_opacity * (0.7 + 0.3 * sin(time * speed + phase))
```
Проблема: выглядит механически. Все звёзды мерцают одинаково.

**Уровень 2 — Хроматическое мерцание (рекомендуется):**
Каждый RGB-канал варьируется на своей частоте, имитируя атмосферную дисперсию:
```glsl
float r = 0.8 + 0.2 * sin(uTime * 1.1 + phase);
float g = 0.8 + 0.2 * sin(uTime * 1.4 + phase + 1.0);
float b = 0.8 + 0.2 * sin(uTime * 0.8 + phase + 2.0);
gl_FragColor = vec4(baseColor.rgb * vec3(r, g, b), alpha);
```

**Уровень 3 — Стохастическая сцинтилляция (продвинутый):**
Шум-вариация яркости с периодическими яркими вспышками:
```glsl
float scint = snoise(vec2(starId * 100.0, uTime * 0.5));
float brightness = 1.0 + scint * 0.4;
// Периодическая яркая вспышка (верхние 5% диапазона шума)
brightness += max(0.0, scint - 0.8) * 3.0;
```

**Визуальный эффект:** Средний (но непропорционально добавляет «жизни» сцене)
**Стоимость для GPU:** Ничтожная (математика в шейдере)
**Сложность:** Простая (уровни 1-2) / Умеренная (уровень 3)
**Лучше всего для:** Каждый звёздный слой. Безусловно необходимо для реалистичности.

## 2. Туманности и газовые облака

### 2.1 WebGL фрагментный шейдер туманности (fBm + Domain Warping) ★ МАКСИМАЛЬНЫЙ ЭФФЕКТ

**Что это:** Полноэкранный GLSL-фрагментный шейдер, генерирующий процедурные облака туманности с помощью фрактального броуновского движения (fBm) с деформацией координатного пространства (domain warping). Это ЕДИНСТВЕННАЯ техника с самым высоким влиянием на реалистичность космоса.

**Как реализовать:**

**Ядро алгоритма — fBm:**
```glsl
float fbm(vec2 p) {
    float value = 0.0;
    float amplitude = 0.5;
    float frequency = 1.0;
    for (int i = 0; i < 6; i++) {  // 6 октав
        value += amplitude * snoise(p * frequency);
        frequency *= 2.0;   // лакунарность
        amplitude *= 0.5;   // усиление
    }
    return value;
}
```

**Domain warping (магический ингредиент — создаёт органичные формы облаков):**
```glsl
float warpedNebula(vec2 uv) {
    vec2 q = vec2(fbm(uv), fbm(uv + vec2(5.2, 1.3)));
    vec2 r = vec2(fbm(uv + 4.0 * q + vec2(1.7, 9.2) + 0.15 * uTime),
                  fbm(uv + 4.0 * q + vec2(8.3, 2.8) + 0.126 * uTime));
    return fbm(uv + 4.0 * r);
}
```

**Цветовое отображение для реалистичных типов туманностей:**
```glsl
// Эмиссионная туманность (H-alpha красная/розовая)
vec3 emissionColor = mix(vec3(0.6, 0.1, 0.15), vec3(0.9, 0.3, 0.4), density);

// Отражательная туманность (рассеянный синий)
vec3 reflectionColor = mix(vec3(0.1, 0.15, 0.4), vec3(0.3, 0.5, 0.9), density);

// Планетарная туманность (OIII бирюзовый/зелёный)
vec3 planetaryColor = mix(vec3(0.05, 0.3, 0.3), vec3(0.2, 0.8, 0.7), density);
```

**Способы интеграции:**
1. **Библиотека shader-web-background** — рендерит GLSL как полноэкранный фон, совместима с Shadertoy, `z-index: -9999`, CSS-фоллбэк. Минимальный размер.
2. **React Three Fiber fullscreen quad** — `<mesh>` с `<planeGeometry args={[2,2]}/>` и кастомным `shaderMaterial`. Интегрируется с пайплайном постобработки R3F.
3. **Компонент shadcn Nebula** — готовый React/Next.js компонент на WebGL fBm. Установка через `shadcn@latest add`. Настраиваемые цвета, плотность, мобильные настройки.
4. **drei `shaderMaterial`** — фабричная функция с автоуправлением uniform-переменными как пропсами. Поддержка hot-reload через `.key`.

**Визуальный эффект:** ВАУ
**Стоимость для GPU:** Низкая-Средняя (только фрагментный шейдер, без геометрии. 6-октавный fBm на современном GPU: ~2мс при 1080p. На мобильных уменьшить до 4 октав)
**Сложность:** Сложная (писать с нуля) / Умеренная (использовать библиотеку/портировать с Shadertoy)
**Лучше всего для:** Главный визуальный дифференциатор. Делает фон объёмным и живым.

**Ключевые ссылки:**
- Глава fBm из The Book of Shaders: https://thebookofshaders.com/13/
- 4-слойный пайплайн wwwtyro: https://wwwtyro.net/2016/10/22/2D-space-scene-procgen.html
- Коллекция туманностей Shadertoy: https://www.shadertoy.com/results?query=tag%3Dnebula
- shader-web-background: https://github.com/xemantic/shader-web-background
- shadcn Nebula: https://www.shadcn.io/shaders/nebula

### 2.2 SVG-фильтр туманности (feTurbulence + feColorMatrix)

**Что это:** Использование встроенного SVG-фильтра `feTurbulence` для генерации шума Перлина, затем цветовое преобразование через `feColorMatrix` и `feComponentTransfer` для создания текстур, похожих на туманность. Чисто декларативно, без JavaScript.

**Как реализовать:**
```html
<svg width="0" height="0">
  <filter id="nebula" x="0%" y="0%" width="100%" height="100%">
    <!-- Базовая турбулентность -->
    <feTurbulence type="fractalNoise" baseFrequency="0.003" numOctaves="5"
                  seed="42" stitchTiles="stitch" result="noise"/>
    <!-- Смещение в цвета туманности -->
    <feColorMatrix type="matrix" in="noise" result="colored"
      values="0   0   0   0   0.15
              0.5 0   0   0   0.05
              0   0   0.8 0   0.2
              0   0   0   0.6 0"/>
    <!-- Усиление контраста -->
    <feComponentTransfer result="final">
      <feFuncR type="gamma" amplitude="1.5" exponent="3" offset="0"/>
      <feFuncG type="gamma" amplitude="1.2" exponent="3" offset="0"/>
      <feFuncB type="gamma" amplitude="2.0" exponent="2" offset="0"/>
      <feFuncA type="gamma" amplitude="1" exponent="2" offset="0"/>
    </feComponentTransfer>
  </filter>
</svg>
<div style="filter: url(#nebula); width: 100vw; height: 100vh; position: fixed;"/>
```

**Плюсы:** Ноль JavaScript, работает без canvas/WebGL, можно анимировать изменением атрибута `seed`
**Минусы:** Меньше контроля чем GLSL, нельзя делать domain warping, производительность различается по браузерам (Safari исторически медленно со сложными SVG-фильтрами), ограничено простыми шумовыми паттернами

**Визуальный эффект:** Средний
**Стоимость для GPU:** Низкая (статично) / Средняя (анимировано)
**Сложность:** Умеренная
**Лучше всего для:** Лёгкий акцент туманности когда WebGL недоступен или избыточен. Хорошо для тонкой фоновой дымки.

### 2.3 CSS слоистая градиентная туманность (текущий подход — путь улучшения)

**Что это:** Ваш текущий подход с `radial-gradient` орбами. Несколько перекрывающихся градиентов с `mix-blend-mode` создают эффект смешивания цветов.

**Как улучшить текущую реализацию:**
- **Добавить режимы наложения** — `mix-blend-mode: screen` на слоях туманности делает цвета аддитивными (свет складывается со светом, как у реальных туманностей). Сейчас не используются режимы наложения.
- **Использовать `conic-gradient`** для хвостов туманности с эффектом развёртки
- **Наложить 5-7 градиентов** вместо 3, с гораздо меньшей индивидуальной прозрачностью (0.08-0.15)
- **Разделение цветов** — расположить тёплые (красные/розовые) и холодные (синие/бирюзовые) градиенты в противоположных позициях. У реальных туманностей цветовые зоны от разных элементов.
- **Добавить `backdrop-filter: blur()`** на промежуточные слои для глубины
- **Анимировать позиции через CSS transforms** (GPU-композитинг) а не изменения свойств

**Улучшенная цветовая палитра для орбов туманности:**
```css
/* Эмиссионная область (H-alpha) */
radial-gradient(ellipse at 30% 40%, rgba(180, 50, 80, 0.12), transparent 60%)

/* Отражательная область */
radial-gradient(ellipse at 70% 30%, rgba(60, 100, 200, 0.10), transparent 55%)

/* OIII эмиссия */
radial-gradient(ellipse at 50% 70%, rgba(40, 180, 160, 0.08), transparent 50%)

/* Горячий водород */
radial-gradient(ellipse at 25% 55%, rgba(200, 80, 120, 0.06), transparent 45%)

/* Глубокое синее рассеивание */
radial-gradient(ellipse at 80% 60%, rgba(30, 50, 140, 0.10), transparent 65%)
```

**Визуальный эффект:** Средний (с улучшениями: Средний-Высокий)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Базовый слой туманности под шейдерной туманностью, или самостоятельно для лёгких страниц

### 2.4 Пре-рендер изображений туманностей с параллаксом

**Что это:** Высокого разрешения фотографии туманностей (NASA/Hubble) или процедурно сгенерированные текстуры, размещённые как фоновые изображения с параллакс-прокруткой.

**Как реализовать:**
- Сгенерировать текстуры через wwwtyro space-2d (https://github.com/wwwtyro/space-2d) — свободная лицензия, процедурная генерация
- Или использовать NASA-изображения из общественного достояния (https://images.nasa.gov)
- Отдавать в WebP, сильно сжатые (quality 60-70 приемлемо для фоновых туманностей)
- Применить через `background-image` с `background-attachment: fixed` или параллакс-трансформами
- Использовать `<picture>` с srcset для адаптивных размеров (2048px десктоп, 1024px планшет, 512px мобильные)
- Применить `filter: blur(1-2px)` и `opacity: 0.3-0.5`, чтобы изображение работало как атмосфера, а не обои

**Визуальный эффект:** Средний-Высокий (зависит от качества исходного изображения)
**Стоимость для GPU:** Низкая (после загрузки). Внимание: большие изображения = медленная начальная загрузка
**Сложность:** Простая
**Лучше всего для:** Быстрое визуальное улучшение, если нет времени на разработку шейдеров. Комбинируйте с процедурными элементами для лучшего результата.

## 3. Космическая пыль и частицы

### 3.1 Плавающие микрочастицы с броуновским движением

**Что это:** Сотни крошечных (0.5-1.5px) частиц, медленно дрейфующих со случайными изменениями направления, имитирующих космическую пыль, освещённую звёздным светом.

**Как реализовать:**
- В WebGL: добавить вторую систему `Points` с очень малым `gl_PointSize` (1-3px)
- Каждая частица имеет вектор скорости, получающий небольшое случайное возмущение каждый кадр (броуновское движение): `velocity += (noise(position * 0.01 + time) - 0.5) * 0.001`
- Очень низкая прозрачность (0.1-0.3), тёплый белый или бледно-золотой цвет
- В Canvas 2D: рисовать 200-500 точек, обновлять позиции через simplex noise-вектор скорости
- В CSS: использовать `@keyframes` с многоточечными путями (`translate3d`) и различными длительностями (30-60 секунд)

**Визуальный эффект:** Средний (тонкий, но необходимый для ощущения «обитаемости»)
**Стоимость для GPU:** Низкая (крошечные частицы, простая математика)
**Сложность:** Простая
**Лучше всего для:** Заполнение «пустоты» между звёздами. Добавляет ощущение масштаба.

---

### 3.2 Пылевые полосы (тёмные затемняющие области) ★ НЕДООЦЕНЁННАЯ ТЕХНИКА

**Что это:** Тёмные, полупрозрачные области, частично блокирующие фоновые звёзды и туманности. В реальном космосе есть тёмные молекулярные облака, блокирующие свет — именно они придают глубину снимкам Хаббла.

**Как реализовать:**
- **CSS-подход:** Крупные элементы с тёмными радиальными градиентами (`rgba(5, 8, 15, 0.3-0.6)`) размещённые между слоями звёзд и туманности по z-порядку
- **Шейдерный подход:** В шейдере туманности инвертировать некоторые шумовые области, чтобы они затемняли вместо осветления:
  ```glsl
  float dust = fbm(uv * 2.0 + vec2(3.14, 1.59));
  dust = smoothstep(0.3, 0.7, dust);
  color *= mix(1.0, 0.2, dust); // затемнение где пыль плотная
  ```
- **Форма:** Неправильная, вытянутая, с мягкими краями. Не круглая.
- Разместить 2-3 крупные пылевые полосы по viewport

**Визуальный эффект:** Средний (но КОЛОССАЛЬНО улучшает воспринимаемую глубину)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Недостающий элемент в большинстве космических фонов. Это то, что отделяет «цветной градиент» от «реального космоса». Темнота между объектами так же важна, как и яркие объекты.

---

### 3.3 Падающие звёзды / метеоры

**Что это:** Редкие, быстрые, яркие полосы, появляющиеся 1-2 раза в минуту. Создают «моменты событий», которые делают фон живым.

**Как реализовать:**
- Спавнить метеор каждые 20-60 секунд (случайный интервал)
- Свойства: случайная стартовая позиция (верх/бока экрана), угол (примерно 30-60 градусов от вертикали), скорость (пересекает экран за 0.3-0.8 секунды), длина (50-200px хвост)
- **Canvas/WebGL:** Рисовать линию с градиентной прозрачностью (яркая голова, затухающий хвост). Использовать аддитивное наложение.
- **CSS:** Анимировать маленький элемент через `transform: translate()` + `rotate()` с градиентным фоном. Применить `filter: blur(0.5px)` для свечения.
- Цвет: бело-голубой с лёгким тёплым оранжевым в хвосте
- Критически важно: сделать их РЕДКИМИ. Частота > 1 за 15 секунд разрушает эффект.

**Визуальный эффект:** Средний (создаёт запоминающиеся моменты)
**Стоимость для GPU:** Ничтожная (один элемент, короткое время жизни)
**Сложность:** Простая
**Лучше всего для:** Слой финальной полировки. Деталь, которую посетители запоминают.

---

### 3.4 Светящаяся космическая пыль (частицы, ловящие свет)

**Что это:** Несколько десятков чуть более крупных частиц (2-5px) с тонкими ореолами свечения, представляющих пылинки, ловящие свет от ближайшей звезды. Ярче микрочастиц, меньше по количеству.

**Как реализовать:**
- Рендерить как точки с радиальным градиентным спрайтом: яркое ядро, затухающее к прозрачности
- Разместить рядом с самой яркой звездой или источником света
- Очень медленный дрейф (медленнее микрочастиц)
- Периодические колебания яркости (как будто переворачиваются и ловят свет под разными углами)
- В WebGL: использовать `AdditiveBlending` для естественного накопления света
- В Canvas: использовать `ctx.globalCompositeOperation = 'lighter'`

**Визуальный эффект:** Низкий-Средний
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Добавление тонкой светимости рядом с яркими элементами. Хорошо работает вокруг компонента луны.

---

## 4. Глубина и параллакс

### 4.1 Многослойный параллакс мыши ★ ОБЯЗАТЕЛЬНО

**Что это:** Разные фоновые слои сдвигаются с разной скоростью в ответ на движение мыши, создавая убедительную иллюзию глубины. Самый глубокий слой едва двигается; ближайший заметно сдвигается.

**Как реализовать:**
```tsx
// Отслеживание позиции мыши, нормализованной к -1...1
const [mouse, setMouse] = useState({ x: 0, y: 0 })

useEffect(() => {
  const handler = (e: MouseEvent) => {
    setMouse({
      x: (e.clientX / window.innerWidth - 0.5) * 2,
      y: (e.clientY / window.innerHeight - 0.5) * 2
    })
  }
  window.addEventListener('mousemove', handler)
  return () => window.removeEventListener('mousemove', handler)
}, [])

// Применить к слоям с разными множителями
// Глубокие звёзды:     translate(mouse.x * 2px,  mouse.y * 2px)   — еле двигается
// Средние звёзды:      translate(mouse.x * 5px,  mouse.y * 5px)
// Ближние звёзды:      translate(mouse.x * 12px, mouse.y * 12px)  — двигается больше всего
// Туманность:          translate(mouse.x * 3px,  mouse.y * 3px)
// Пыль:                translate(mouse.x * 8px,  mouse.y * 8px)
```

**Критические детали:**
- Сгладить движение через lerp: `current += (target - current) * 0.05` на кадр
- Использовать `transform: translate3d()` (GPU-композитинг, без layout thrash)
- ТОНКОСТЬ: максимальный сдвиг 10-20px для самого глубокого, 30-50px для ближайшего. Больше — вызывает укачивание.
- Полностью отключить для `prefers-reduced-motion`
- Использовать `will-change: transform` на параллакс-слоях

**С Framer Motion:**
```tsx
const mouseX = useMotionValue(0)
const mouseY = useMotionValue(0)
const smoothX = useSpring(mouseX, { stiffness: 50, damping: 20 })
const smoothY = useSpring(mouseY, { stiffness: 50, damping: 20 })

// Для каждого слоя
const layerX = useTransform(smoothX, v => v * depthMultiplier)
const layerY = useTransform(smoothY, v => v * depthMultiplier)
```

**Визуальный эффект:** Высокий (самый простой способ добавить ощущение глубины)
**Стоимость для GPU:** Низкая (только трансформы)
**Сложность:** Простая
**Лучше всего для:** Каждый космический фон. Безусловно необходимо.

---

### 4.2 CSS Perspective + translateZ (настоящая 3D-глубина)

**Что это:** Использование CSS `perspective` на контейнере и `translateZ` на дочерних слоях для создания математически корректного параллакса без JavaScript.

**Как реализовать:**
```css
.space-container {
  perspective: 1000px;
  perspective-origin: 50% 50%;
  overflow: hidden;
  height: 100vh;
}

.star-layer-deep {
  transform: translateZ(-500px) scale(1.5); /* Далеко, увеличен для компенсации */
}

.star-layer-mid {
  transform: translateZ(-200px) scale(1.2);
}

.star-layer-near {
  transform: translateZ(0px); /* Ближайший */
}
```

**Плюсы:** Чистый CSS, математически корректный параллакс, GPU-композитинг
**Минусы:** Может вызвать проблемы с `position: fixed` дочерними элементами, нужна аккуратная компенсация масштаба
**Лучше комбинировать с:** scroll-based translateY для параллакса прокрутки

**Визуальный эффект:** Средний-Высокий
**Стоимость для GPU:** Ничтожная (GPU-композитинг)
**Сложность:** Умеренная (правильная математика)
**Лучше всего для:** Параллакс прокрутки, где нужна «бесплатная» глубина без JavaScript-анимационных циклов

---

### 4.3 Атмосферная перспектива (глубинный туман)

**Что это:** Удалённые объекты выглядят более голубыми/блёклыми, ближние — чётче и насыщеннее. Так наши глаза воспринимают глубину, и это работает для космоса тоже.

**Как реализовать:**
- **Послойный подход (CSS):** Глубокие слои получают `opacity: 0.3-0.5` и `filter: blur(0.5-1px)` и сдвиг в синий цвет
- **В WebGL:** Использовать Three.js `Fog` или `FogExp2` с тёмно-синим цветом, совпадающим с фоном:
  ```tsx
  <fog attach="fog" args={['#0a0e1a', 50, 500]} />
  ```
- **В шейдерах:** Смешивать цвет звезды к цвету фона в зависимости от глубины:
  ```glsl
  float fogFactor = smoothstep(nearPlane, farPlane, depth);
  color = mix(starColor, fogColor, fogFactor);
  ```
- **Цветовой сдвиг:** Далёкие звёзды должны быть чуть голубоватыми (#8890aa), ближние сохраняют настоящий цвет

**Визуальный эффект:** Высокий (мозг автоматически читает «3D-пространство»)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Каждая слоистая система. Применять повсеместно.

---

### 4.4 Имитация глубины резкости

**Что это:** Лёгкое размытие самого глубокого и опционально ближайшего слоя, фокусирующее взгляд на середине (где живёт контент).

**Как реализовать:**
- **CSS:** `filter: blur(1px)` на глубоком звёздном слое, `blur(0.5px)` на ближней плавающей пыли
- **React Postprocessing:**
  ```tsx
  <EffectComposer>
    <DepthOfField focusDistance={0.5} focalLength={0.02} bokehScale={2} />
  </EffectComposer>
  ```
- **Важно:** Не размывать всё. Средний план должен быть чётким. Это создаёт «плоскость фокуса», которая естественно направляет взгляд на контент.

**Визуальный эффект:** Средний (тонкий, но добавляет кинематографичность)
**Стоимость для GPU:** Низкая (CSS blur) / Средняя (WebGL DoF)
**Сложность:** Простая (CSS) / Умеренная (WebGL)
**Лучше всего для:** В комбинации с другими индикаторами глубины.

---

### 4.5 Масштабные индикаторы глубины

**Что это:** Варьирование размеров частиц для подразумевания расстояния. Не случайно — систематически более крупные частицы в «ближних» слоях, мельче в «глубоких».

**Как реализовать:**
- Глубокий слой: 0.3-0.8px звёзды, очень тусклые
- Средний слой: 0.8-2.0px звёзды, умеренная яркость
- Ближний слой: 1.5-3.5px звёзды, яркие, с ореолами свечения
- Особые звёзды: 4-8px с выраженным свечением (всего 2-5 штук)

Это частично есть в вашем starfield, но может быть более драматичным. СООТНОШЕНИЕ РАЗМЕРОВ между слоями важнее абсолютных размеров. Глубокие звёзды КРОШЕЧНЫЕ (еле видны), особые звёзды КРУПНЫЕ.

**Визуальный эффект:** Средний
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Усиление параллаксной глубины. Всегда использовать вместе с параллаксом.

---

## 5. Освещение и атмосфера

### 5.1 Согласованность глобального источника света

**Что это:** Установление единого доминирующего направления света, которое уважают все элементы. Ваша луна уже имеет освещение сверху-слева. Все остальные элементы должны соответствовать.

**Как реализовать:**
- Определить вектор направления света: например, `lightDir = normalize(vec3(-0.3, 0.5, 0.7))` (верхний-левый-передний)
- Туманность: ярче с верхней-левой стороны, темнее с нижней-правой
- Градиентные орбы: расположить тёплые блики сверху-слева, холодные тени снизу-справа
- Пылевые частицы рядом с источником света выглядят ярче
- Свечение луны должно исходить преимущественно со стороны направления света
- В CSS: все `radial-gradient` орбы и свечения должны быть позиционированы согласованно с направлением света

**Визуальный эффект:** Средний (но его отсутствие ОЧЕНЬ заметно)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая (осознанное дизайн-решение, не код)
**Лучше всего для:** Визуальная согласованность. Сейчас ваши градиентные орбы расположены несколько хаотично — выравнивание их с источником света луны объединит сцену.

---

### 5.2 Эффекты бликов линзы

**Что это:** Яркие звёзды или источник света, создающие характерные шестиугольные/круглые световые артефакты. Распространены в космической фотографии.

**Как реализовать:**
- **CSS-подход:** Несколько `radial-gradient` кругов уменьшающегося размера и прозрачности, расположенных вдоль линии от источника света через центр экрана
- **Canvas/WebGL:** Нарисовать серию полупрозрачных кругов с аддитивным наложением вдоль оси блика
- **Спрайтовый подход:** Использовать готовую текстуру блика (прозрачный PNG) с `mix-blend-mode: screen`
- **R3F:** В `@react-three/drei` есть эффекты бликов линзы
- Сделать ТОНКО: космический блик должен быть едва заметен, а не как у J.J. Abrams

**Визуальный эффект:** Низкий-Средний
**Стоимость для GPU:** Низкая
**Сложность:** Простая
**Лучше всего для:** Один тонкий блик рядом с самым ярким элементом. Легко переборщить.

---

### 5.3 Объёмный свет (лучи Бога)

**Что это:** Лучи света, исходящие от яркого источника, рассеивающиеся через пыль/газ. Создаёт драматичные световые полосы.

**Как реализовать:**
- **CSS-подход:** `conic-gradient` с центром на источнике света с секторами очень низкой прозрачности:
  ```css
  background: conic-gradient(
    from 200deg at 80% 20%,
    transparent 0deg,
    rgba(125, 211, 252, 0.02) 15deg,
    transparent 30deg,
    rgba(125, 211, 252, 0.015) 50deg,
    transparent 65deg,
    ...
  );
  ```
- **WebGL:** Постобработка радиальным размытием от позиции источника света (дорого но потрясающе)
- **Шейдер:** Сэмплировать вдоль лучей от фрагмента к источнику, накапливая плотность из функции шума

**Визуальный эффект:** Высокий (при тонкости) / Вау (при заметности)
**Стоимость для GPU:** Низкая (CSS) / Высокая (WebGL постобработка)
**Сложность:** Простая (CSS) / Сложная (WebGL)
**Лучше всего для:** Драматичные Hero-секции. Использовать экономно — максимум один источник света.

---

### 5.4 Эффект виньетки ★ ОБЯЗАТЕЛЬНО

**Что это:** Затемнение краёв и углов viewport, фокусирующее внимание на центре. Каждая хорошая космическая сцена имеет это.

**Как реализовать:**
- **CSS (самый простой и лучший):**
  ```css
  .vignette {
    position: fixed;
    inset: 0;
    pointer-events: none;
    background: radial-gradient(
      ellipse at 50% 50%,
      transparent 40%,
      rgba(5, 8, 15, 0.4) 70%,
      rgba(5, 8, 15, 0.8) 100%
    );
    z-index: /* над звёздами, под контентом */;
  }
  ```
- **React Postprocessing:**
  ```tsx
  <Vignette eskil={false} offset={0.1} darkness={1.1} />
  ```
- Настроить стопы градиента так, чтобы центральные 40% были чистыми, края затухали до почти чёрного
- Эллиптическая форма (не круг) естественно соответствует пропорциям viewport

**Визуальный эффект:** Высокий (массово улучшает фокус и восприятие глубины)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** ВСЕГДА. Каждый космический фон нуждается в виньетке. Это самый дешёвый апгрейд с «вау»-эффектом.

---

### 5.5 Цветокоррекция (Color Grading)

**Что это:** Применение единой цветовой температуры ко всем элементам, чтобы они выглядели как часть одной среды.

**Как реализовать:**
- **CSS `filter` на обёртке:**
  ```css
  filter: saturate(0.85) brightness(0.95) contrast(1.05);
  ```
- Определить мастер-палитру и убедиться, что все элементы от неё производны:
  - Фон: `#0a0e1a` (почти чёрный тёмно-синий) — хорошо, оставить
  - Холодный акцент: `#7dd3fc` → сдвинуть к `#6bc5f0` (менее насыщенному)
  - Тёплый акцент: добавить `#ffd4a0` (тёплый звёздный/пылевой акцент)
  - Тёплая туманность: `#8b3a5c` (приглушённая роза, не неон-розовый)
  - Холодная туманность: `#2a4a7f` (глубокий синий, не электрический)
- **Ключевой инсайт:** Всё должно иметь лёгкий сине-зелёный ПОДТОН, согласованный с базой #0a0e1a. Даже тёплые элементы должны ощущаться как «холодно-тёплые», а не «земно-тёплые».

**Визуальный эффект:** Средний (устраняет ощущение «коллажа»)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая (дисциплина дизайна)
**Лучше всего для:** Финальный проход полировки после размещения всех элементов.

---

## 6. Цветовая наука для космоса

### 6.1 Как избежать «мультяшного космоса»

Ошибка №1: перенасыщенные синие и фиолетовые. Реальный космос преимущественно ЧЁРНЫЙ с тонким цветом.

**Правила:**
1. **90% viewport должно быть почти чёрным** (диапазон #0a0e1a). Цвет существует только в конкретных регионах.
2. **Цвета туманности никогда не должны превышать 15-20% непрозрачности** на фоне. Если видно, что это градиент — слишком сильно.
3. **Никогда не использовать чистые оттенки.** `hsl(240, 100%, 50%)` НЕ существует в космосе. Десатурировать всё на 30-50%.
4. **Тёплое и холодное ВМЕСТЕ.** Реальные туманности имеют и эмиссионные (тёплые красные/розовые) и отражательные (холодные синие) регионы, часто рядом. Только холодные цвета = мультяшность.
5. **Цвет фона — доминирующий цвет.** Всё остальное — акцент.

### 6.2 Реалистичные цветовые палитры космоса

**NASA/Hubble «Палитра Хаббла» (SHO-маппинг):**
- S-II (сера): Красный, маппится на Красный канал
- H-alpha (водород): Зелёно-розовый, маппится на Зелёный канал
- O-III (кислород): Сине-бирюзовый, маппится на Синий канал
- Результат: культовый вид «Столпов Творения»

**Для веб-реализации, аппроксимировать:**
```

Глубокий тёмно-синий:      #0f1729, #121d33
Красный туманности (H-alpha): #5c2233, #8b3a4f, #a34460 (приглушённый!)
Синий туманности (OIII):     #1a3355, #2a4a7f, #3d6baa
Бирюзовый туманности:       #1a4040, #2a6b6b, #3d8a8a

Пыль (затемняющая):         #060810, #0a0d14 (ещё темнее фона)
```

### 6.3 Смешивание цветовых температур

**Правило:** Источники света ТЁПЛЫЕ (звёзды излучают). Отражённый/рассеянный свет ХОЛОДНЫЙ (отражение туманности). Тёмные области НЕЙТРАЛЬНО-ХОЛОДНЫЕ (поглощение пылью).

- Звёзды: тёплое белое до жёлто-оранжевого ядро, сине-белое свечение только для O/B типов
- Эмиссия туманности: тёплая (красная, розовая, оранжевая) — это газ, нагретый звёздами
- Отражение туманности: холодное (синее, бирюзовое) — это пыль, отражающая звёздный свет
- Фоновое свечение: холодный синий подтон
- Пылевые полосы: почти чёрные со слегка тёплым подтоном (тёмный коричнево-чёрный, не сине-чёрный)

### 6.4 Воспроизведение ощущения цвета снимков Хаббла

NASA фальш-окрашивает свои изображения. «Вид Хаббла» создаётся за счёт:
1. **Высокий динамический диапазон, сжатый в видимый** — одновременно видны и очень яркие звёзды, и тусклые туманности
2. **Маппинг конкретных эмиссионных линий** (SHO → RGB или NB → RGB) создающий комбинации бирюзовый+оранжевый+розовый
3. **Жёсткие кривые контраста** — тени вдавлены в чисто чёрный, хайлайты яркие но не пересвечены
4. **Зерно/шум** — реальные снимки имеют шум детектора. Тонкий шумовой оверлей добавляет аутентичности.

**Чтобы воспроизвести:** Применить тонкий `noise` эффект постобработки (2-4% непрозрачности), вдавить тени через CSS `contrast(1.1)`, убедиться что хайлайты остаются ниже чисто белого.

// ПРАВИЛЬНО: Проверять видимость перед анимацией

### 7.4 Обработка prefers-reduced-motion ★ ОБЯЗАТЕЛЬНО

```tsx
// Хук
function useReducedMotion() {
  const [reduced, setReduced] = useState(false)
  useEffect(() => {
    const mq = window.matchMedia('(prefers-reduced-motion: reduce)')
    setReduced(mq.matches)
    const handler = (e: MediaQueryListEvent) => setReduced(e.matches)
    mq.addEventListener('change', handler)
    return () => mq.removeEventListener('change', handler)
  }, [])
  return reduced
}

// Использование: когда уменьшенное движение предпочтительно
// - Остановить мерцание всех звёзд
// - Остановить параллакс мыши
// - Остановить плавающие/дрейфующие анимации
// - Показать статичный снимок туманности (заморозить время шейдера)
// - Оставить: виньетку, статичные звёзды, статичные цвета туманности (это не движение)
// - Падающие звёзды: полностью отключить
```

**Framer Motion:** Использовать хук `useReducedMotion()` из `motion/react` — возвращает boolean.

### 7.5 Стратегия производительности для мобильных

**Упрощать агрессивно:**
- Количество звёзд: десктоп 5000 → мобильные 1500
- Октавы шейдера туманности: десктоп 6 → мобильные 3-4
- Отключить параллакс мыши (нет мыши)
- Включить параллакс гироскопа (опционально, расход батареи)
- Уменьшить разрешение canvas: `canvas.width = window.innerWidth * Math.min(devicePixelRatio, 2)` — ограничить 2x даже на 3x экранах
- Рассмотреть CSS-only фоллбэк для слабых устройств: определять через `navigator.hardwareConcurrency < 4` или `navigator.deviceMemory < 4`
- Отключить bloom-постобработку
- Использовать `<picture>` с меньшими пре-рендеренными изображениями туманностей вместо real-time шейдеров

### 8.2 Космический зум при скролле

**Что это:** По мере прокрутки страницы звёзды слегка увеличиваются/дрейфуют, создавая ощущение движения через космос.

**Как реализовать с Framer Motion:**
```tsx
const { scrollYProgress } = useScroll()

// Звёзды слегка увеличиваются при скролле (полёт вперёд)
const starScale = useTransform(scrollYProgress, [0, 1], [1, 1.15])

// Туманность слегка затухает (пролетаем мимо)
const nebulaOpacity = useTransform(scrollYProgress, [0, 0.5, 1], [0.3, 0.25, 0.15])

// Параллакс-смещение
const deepLayerY = useTransform(scrollYProgress, [0, 1], [0, -50], { clamp: false })
const nearLayerY = useTransform(scrollYProgress, [0, 1], [0, -200], { clamp: false })
```

**Визуальный эффект:** Средний
**Стоимость для GPU:** Низкая (только трансформы)
**Сложность:** Простая
**Лучше всего для:** Страницы со значительным скролл-контентом. Усиливает ощущение путешествия.

### 8.3 Параллакс по гироскопу (мобильные)

**Что это:** Использование датчиков ориентации устройства для сдвига слоёв на основе наклона телефона.

**Как реализовать:**
```tsx
useEffect(() => {
  if (typeof DeviceOrientationEvent === 'undefined') return

  const handler = (e: DeviceOrientationEvent) => {
    const x = (e.gamma || 0) / 45  // Наклон влево/вправо, нормализован к -1..1
    const y = (e.beta || 0 - 45) / 45  // Наклон вперёд/назад, центрирован на 45°
    setTilt({ x: Math.max(-1, Math.min(1, x)), y: Math.max(-1, Math.min(1, y)) })
  }

  // iOS 13+ требует разрешения
  if (typeof (DeviceOrientationEvent as any).requestPermission === 'function') {
    (DeviceOrientationEvent as any).requestPermission().then((perm: string) => {
      if (perm === 'granted') window.addEventListener('deviceorientation', handler)
    })
  } else {
    window.addEventListener('deviceorientation', handler)
  }

  return () => window.removeEventListener('deviceorientation', handler)
}, [])
```

**Визуальный эффект:** Средний (ощущается магически на мобильных)
**Стоимость для GPU:** Низкая
**Сложность:** Умеренная (разрешения iOS, фильтрация шума сенсоров)
**Лучше всего для:** Улучшение мобильного опыта. Заменяет параллакс мыши.

### 8.4 Стрикинг звёзд при скорости скролла

**Что это:** При быстрой прокрутке звёзды кратковременно вытягиваются в полосы (как при варп-скорости), затем возвращаются к точкам при остановке.

**Как реализовать:**
```tsx
const { scrollY } = useScroll()
const scrollVelocity = useVelocity(scrollY) // пикселей в секунду

// Маппинг скорости в длину полосы
const streakLength = useTransform(scrollVelocity, [-2000, 0, 2000], [15, 0, 15])
// Маппинг скорости в изменение прозрачности звёзд
const streakOpacity = useTransform(scrollVelocity, [-1000, 0, 1000], [0.3, 0, 0.3])
```

В шейдере звёзд, удлинять точки по uniform стрикинга:
```glsl
// Вместо круглых точек, растягиваем по Y
vec2 centered = gl_PointCoord - 0.5;
centered.y /= (1.0 + uStreak * 5.0); // растяжение
float d = length(centered);
```

**Визуальный эффект:** Средний (очень круто когда заметишь)
**Стоимость для GPU:** Низкая
**Сложность:** Умеренная
**Лучше всего для:** Страницы с активной прокруткой. Создаёт ощущение «гиперпространства».

### 8.5 Hover-эффекты на небесных объектах

**Что это:** Декоративная луна или яркие звёзды реагируют на приближение курсора — тонкое усиление свечения, лёгкий масштаб или параллакс-сдвиг.

**Как реализовать:**
```tsx
<motion.div
  whileHover={{ scale: 1.02, filter: 'brightness(1.1)' }}
  transition={{ type: 'spring', stiffness: 200, damping: 20 }}
>
  <DecorativeMoon />
</motion.div>
```

**Визуальный эффект:** Низкий (но добавляет интерактивность)
**Стоимость для GPU:** Ничтожная
**Сложность:** Простая
**Лучше всего для:** Сцена ощущается отзывчивой. Не перебарщивать — один-два интерактивных элемента максимум.

## 10. Комбинирование техник — стек слоёв

### 10.1 Рекомендуемый Z-порядок (от заднего к переднему)

```
Слой 0 (z-0):   CSS background: #0a0e1a (сплошная основа)
Слой 1 (z-1):   Шейдерная/SVG туманность (полноэкранная, низкая прозрачность 0.15-0.25)
Слой 2 (z-2):   Пылевые полосы (тёмные затемняющие области)
Слой 3 (z-3):   Глубокое звёздное поле (крошечные, тусклые, еле двигаются при параллаксе)
Слой 4 (z-4):   Среднее звёздное поле (средние, умеренный параллакс)
Слой 5 (z-5):   CSS-оверлеи туманности (градиентные орбы, очень низкая прозрачность)
Слой 6 (z-6):   Космическая микро-пыль (частицы с броуновским движением)
Слой 7 (z-7):   Ближнее звёздное поле (яркие, сильный параллакс)
Слой 8 (z-8):   Особые звёзды с ореолами свечения (2-5 штук всего)
Слой 9 (z-9):   Оверлей виньетки
Слой 10 (z-10): Декоративная луна + свечение
Слой 11 (z-11): Падающие звёзды (периодические)
Слой 12+:       Контент (текст, UI и т.д.)
```

### 10.2 Ключевые правила композитинга

1. **Всё за контентом использует `pointer-events: none`**
2. **Использовать `position: fixed` для фоновых слоёв** (они не должны скроллиться с контентом)
3. **Разделять статичные и анимированные слои** — статичные могут быть одиночными отрисовками, анимированные требуют rAF
4. **Использовать `mix-blend-mode: screen`** для любого светоизлучающего элемента, перекрывающего другой светоизлучающий элемент (аддитивный свет)
5. **Не превышать 3-4 GPU-композитных слоя** одновременно анимируемых — каждый отдельная текстура в GPU-памяти
6. **WebGL canvas — это ОДИН слой** — все звёзды, пыль и т.д., рендерящиеся внутри, считаются одним композитным слоем

### 10.3 Читаемость текста поверх сложного фона

**Критические техники:**
- **Виньетка естественно помогает** — затемняет края, где часто живёт текст
- **Локальное затемнение подложки:** использовать `backdrop-filter: brightness(0.5) blur(4px)` на контейнерах контента
- **Тень текста:** `text-shadow: 0 0 20px rgba(0,0,0,0.8), 0 0 40px rgba(0,0,0,0.5)` для текста без засветки
- **«Тихие зоны»:** Держать центр-верх (где живёт текст hero) свободным от ярких туманностей/звёзд. Смещать визуальную сложность к краям и углам.
- **Минимальный контраст:** Обеспечить 4.5:1 WCAG AA на ВСЁМ тексте, даже поверх самого загруженного участка фона. Тестировать скриншотами.

### 10.4 Баланс «впечатляющий» vs «отвлекающий»

**Золотое правило:** Фон должен быть замечен в первые 2 секунды, затем забыт, пока посетитель читает контент. Если фон КОНКУРИРУЕТ с контентом — это перебор.

**Практические лимиты:**
- Общая анимация фона не должна занимать более 20% площади viewport в любой момент
- Ни один элемент не должен двигаться быстрее 10px/секунду (кроме падающих звёзд)
- Мерцание должно затрагивать <30% звёзд в любой момент времени
- Параллакс мыши должен быть незаметен при обычной скорости мыши — виден только при намеренном медленном движении
- Если убрать весь текст контента и фон сам по себе выглядит «завершённым» / как арт-работа — вероятно, он слишком перегружен. Фон должен ощущаться НЕПОЛНЫМ без контента.

## Оценка текущей кодовой базы

### Что есть (и что хорошо):
- `StarField.tsx`: 4-слойная canvas-система с базовым мерцанием. Хороший фундамент.
- `GradientOrb.tsx`: 3 плавающих орба. Концепция верная, исполнение требует доработки.
- `MoonGlow.tsx`: Ореолы свечения луны. Работает хорошо.
- `HeroSection.tsx`: Детальная декоративная луна с правильным освещением. Отличная работа.
- `globals.css`: CSS-переменные цветов, анимации. Крепкая база.
- prefers-reduced-motion: Уже обрабатывается в StarField. Хорошо.

### Что отсутствует (причины «плоскости»):
1. **Нет виньетки** — края такой же яркости, как центр
2. **Нет пылевых полос** — нет тёмных областей, создающих контраст глубины
3. **Звёзды слишком однообразны по цвету** — только холодные голубые/белые варианты, нет тёплых звёзд
4. **Нет атмосферной перспективы** — все слои одинаковой чёткости вне зависимости от «расстояния»
5. **Нет параллакса мыши** — нет реакции движения на пользователя, ощущается статичным
6. **Градиентные орбы слишком насыщенные** — голубые и фиолетовые орбы читаются как «цветные пятна» а не «газовая туманность»
7. **Нет режимов наложения** — орбы не взаимодействуют друг с другом или звёздами (нет `mix-blend-mode: screen`)
8. **Нет шума/зернистости** — слишком чисто/цифрово
9. **Нет процедурной текстуры туманности** — градиенты гладкие, реальные туманности имеют фрактальную детализацию
10. **Нет особых звёзд со свечением** — нет «героических» звёзд, притягивающих взгляд
